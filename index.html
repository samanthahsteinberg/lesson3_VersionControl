Highlight #1 About Version Control

People who are not familiar with Git should start off by understanding what version control is. Version control is a system that records changes to a file or group of files over time so that older versions can be recalled later if needed. This can be helpful to graphic designers or web designers to keep all versions of layouts to change around or edit. A Version Control System (VCS) allows for this editing and retrieving older files. There is something called a local version control system, which has a simple database that keeps all edits made to files by different users in one “holding area”. There is also something called a centralized version control system, which is very similar to a local VCS, but this is based on a single server that holds all files, and other users can have access to edit files through the single server. Many people prefer this type of VCS because anyone can access at any time, including multiple people at once. Lastly, there is something called a distributed version control system, in which clients can have the full history of the repository with all files. This combats the issue of the server being down and not being able to access files.

Highlight #2 Git Basics

Git is different from other version control systems in the way that Git processes and organizes data and information. Git stores its information and data as a snapshot. Every time a project is modified and saved, Git takes a sort of picture of what is changed from the last edit, and saves that photo. If a picture is taken and nothing is changed, Git will not store the file again in order to save room. This goes to show how advanced Git is in understanding file changes and modifications. This distinction between how Git stores data and how other VCS store data is the main reason that sets Git apart from competitors.

Highlight #3 Getting a Git Repository

You can get a Git repository in 2 ways: 1. Take a local directory that is not under version control. In order to do this, you have to locate the projects directory. There are different ways to access a directory based on which system you use. Once finished, you have created a new subdirectory linked to Git. In order to track it, you must make an initial commit, which makes your project now linked with Git! The second way to get a Git repository is by cloning an existing one. To do this, you need the command git clone. After doing this, git clone is used and acts as a copy for the new project being implemented.

Highlight #4 Recording Changes to the Repository

In the repository, there should be files that have been edited and saved. There are two types of files in the repository: tracked and untracked. Tracked files are those that have been saved in the last snapshot and stored in Git. Untracked files are those that are not in the Git database. As files get edited by users, Git will see that they have been modified and will then become tracked in Git. It is also important to check the status of files in stored in Git. To check the status, you need to utilize the git status command. If status is good, you will see a chain of commands, which means that there are no issues within the directory. Additionally, you can track new files with the git add command. There are many different ways to understand files and check them within Git.

Highlight #5 Account Setup and Configuration

GitHub is the main repository for all Git holders. Most people who hold a Git repository is on GitHub. This shows that GitHub is a reliable host for important files being stored through Git. In order to start an account, you can sign up through GitHub.com and register for free, by just providing a username and password. After that, you will need to link SSH Access, which can be done through something like Cloud9. After all is linked, you can customize your GitHub account with a different avatar, profile information, etc. GitHub can be used for personal and professional reasons and will keep all files secure and safely stored.
